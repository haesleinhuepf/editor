# Data Management Plan

## Data Description
* Type of data (e.g., quantitative, qualitative, imagery, sensor data)
* Data formats (e.g., .csv, .json, .tiff, .fasta)
* Estimated volume of data (size in GB/TB or number of records)
* Granularity and scale of the data
* Expected frequency of data generation

## Data Collection and Creation
* Methods of collection (e.g., surveys, automated sensors, manual observation)
* Equipment used for data acquisition
* Sampling methods and intervals
* Procedures for data cleaning and preprocessing
* Documentation of experimental protocols

## Data Storage and Preservation
* Short-term storage locations (e.g., local servers, secure cloud storage)
* Long-term preservation plans (e.g., institutional repositories, public archives)
* Backup frequency and verification procedures
* Security measures (e.g., encryption, firewalls, access controls)
* Data migration plans for obsolete formats

## Data Sharing and Access
* Timing of data release (e.g., immediate, after publication, embargo periods)
* Accessibility levels (e.g., open access, restricted to authorized users)
* Licensing agreements (e.g., Creative Commons, proprietary licenses)
* Contact information for data requests
* Procedures for requesting access to sensitive data

## Metadata Standards
* Metadata schema used (e.g., Dublin Core, DDI, Darwin Core)
* Documentation of variables and units
* Provenance tracking (history of transformations)
* Standardized naming conventions for files
* Geographic coordinates or spatial references (if applicable)

## Ethics and Legal Issues
* Informed consent procedures
* Anonymization or de-identification methods
* Intellectual property rights and ownership
* Compliance with regulations (e.g., GDPR, HIPAA, IRB approvals)
* Potential risks to human subjects or sensitive information
